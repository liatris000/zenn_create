補助金の申請書類は、「申請要項を読む → 自社情報を当てはめる → 事務局的な文体で書く」という作業を何十項目分も繰り返す。この単調さと難解さの組み合わせが、書き始める前に手を止める原因になる。

Claude API を使って「会社プロファイル (JSON) × 補助金要件テキスト」から申請書ドラフトを生成するスクリプトを作った。デジタル化・AI 導入補助金のような構造的な申請書には特に相性が良い。実装コードは GitHub で公開している。

## 全体の構成

入力・処理・出力の 3 レイヤーで実装している。

- **入力**: `company_profile.json`（会社概要・事業内容・従業員数等）+ 補助金要件テキスト
- **処理**: Claude API（Messages API）で申請書の各セクションを順番に生成
- **出力**: `output/draft.md` — そのまま申請書のドラフトとして使える Markdown

## 実装の流れ

<!-- Day 2: 実際のコードをここに追加 -->

### 1. 補助金要件のパース

申請様式の各設問をリスト化するプロンプト設計。様式が変わっても対応できるよう、要件テキストをシステムプロンプトで渡す方式にした。

### 2. company_profile スキーマ設計

どの情報をどのフォーマットで持つかを決める。設問との照合がしやすい構造にすることが重要だった。

### 3. セクション別生成ループ

全設問を一括でプロンプトに渡す方法も試したが、長文になるほど Claude の回答精度が落ちた。セクションごとに API を呼ぶ設計に切り替えることで安定した。

### 4. ドキュメント組み立て

生成したセクションを Markdown に結合し、設問番号とラベルを付与して最終ドキュメントを作る。

## データアナリスト視点

補助金申請書類の生成は「非構造化テキスト → 構造化ドキュメント」の変換であり、データ加工の発想と地続きだ。「どの会社情報をどの設問項目に充てるか」のマッピングは ETL で言えば transform ステップそのもので、ここがズレると最終書類が設問と噛み合わない。申請要件を正規化して保持しておけば、次年度の様式変更にも差分だけ対応できる。
