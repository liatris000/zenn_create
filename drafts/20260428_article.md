# Claude CodeのHooksで「タスク完了をChatworkに自動通知」を30分で作った

---

## はじめに

Claude Codeに長めのタスクを頼むとき、「終わったかな？」と何度も画面を確認しにいく時間、地味にもったいないと思いませんか。

社内でも「Claudeが作業してる間、他の仕事できたらいいのに」という声があって、じゃあ試してみようということで、**Claude Code の Stop Hook を使って、タスク完了時に自動でChatworkに通知が飛ぶ仕組み**を作ってみました。

結論から言うと、**実装自体は30分以下**。実際に席を外しながら作業を任せられるようになって、地味ですが毎日ストレスが減っています。

---

## やったこと

### 1. Claude Code の「Hooks」って何？

Claude Code には、特定のイベントが発生したときにシェルコマンドを自動実行できる **Hooks** 機能があります（2025年半ばに正式リリース）。

イベントの種類はいくつかあって、今回使うのは **`Stop`** ——Claude Codeがタスクを終えて停止したタイミングで発火するやつです。

仕組みはシンプルで：

1. Claude Code がタスク完了（Stop）
2. `.claude/settings.json` に書いたフックが発火
3. 指定したシェルスクリプトが実行される
4. スクリプトが Chatwork API を叩いて通知を送る

それだけです。

### 2. ファイル構成

作ったのは2ファイルだけです。

```
プロジェクト/
  ├── .claude/
  │   ├── settings.json   ← フック設定
  │   └── hooks/
  │       └── notify_chatwork.sh   ← 通知スクリプト
```

### 3. フック設定（settings.json）

`.claude/settings.json` に以下を書くだけです。

```json
{
  "hooks": {
    "Stop": [
      {
        "matcher": "",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/notify_chatwork.sh"
          }
        ]
      }
    ]
  }
}
```

`matcher` を空にすると**全タスクで発火**します。特定のツール実行後だけに絞りたいときは正規表現で指定できます。

### 4. 通知スクリプト（notify_chatwork.sh）全文

```bash
#!/bin/bash
# Claude Code Stop Hook → Chatwork 通知スクリプト

set -euo pipefail

CHATWORK_API_TOKEN="${CHATWORK_API_TOKEN:-}"
CHATWORK_ROOM_ID="${CHATWORK_ROOM_ID:-}"
NOTIFY_TO_ACCOUNT_ID="${NOTIFY_TO_ACCOUNT_ID:-}"

if [[ -z "$CHATWORK_API_TOKEN" || -z "$CHATWORK_ROOM_ID" ]]; then
  echo "[notify_chatwork] CHATWORK_API_TOKEN / CHATWORK_ROOM_ID が未設定のためスキップ" >&2
  exit 0
fi

# Claude Code が Stop フック経由で渡す JSON を stdin から受け取る
HOOK_INPUT="$(cat)"

SESSION_ID=$(echo "$HOOK_INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('session_id','unknown'))" 2>/dev/null || echo "unknown")
STOP_REASON=$(echo "$HOOK_INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('stop_reason','end_turn'))" 2>/dev/null || echo "end_turn")
TRANSCRIPT_PATH=$(echo "$HOOK_INPUT" | python3 -c "import sys,json; d=json.load(sys.stdin); print(d.get('transcript_path',''))" 2>/dev/null || echo "")

# 直前のAssistantメッセージ（最後の要約）を取得
LAST_MESSAGE="（要約取得できず）"
if [[ -n "$TRANSCRIPT_PATH" && -f "$TRANSCRIPT_PATH" ]]; then
  LAST_MESSAGE=$(python3 - "$TRANSCRIPT_PATH" <<'PYEOF'
import sys, json
path = sys.argv[1]
try:
    with open(path) as f:
        messages = json.load(f)
    for msg in reversed(messages):
        if msg.get("role") == "assistant":
            content = msg.get("content", "")
            if isinstance(content, str) and content.strip():
                print(content[:200])
                break
            elif isinstance(content, list):
                for block in reversed(content):
                    if block.get("type") == "text":
                        print(block.get("text","")[:200])
                        break
                break
except Exception as e:
    print(f"（取得エラー: {e}）")
PYEOF
)
fi

# 絵文字マッピング
case "$STOP_REASON" in
  "end_turn")   EMOJI="✅"; STATUS="完了" ;;
  "tool_use")   EMOJI="🔧"; STATUS="ツール実行" ;;
  "max_tokens") EMOJI="⚠️"; STATUS="トークン上限" ;;
  *)            EMOJI="🤖"; STATUS="停止（${STOP_REASON}）" ;;
esac

NOW=$(date '+%Y-%m-%d %H:%M:%S')
CWD=$(pwd)

if [[ -n "$NOTIFY_TO_ACCOUNT_ID" ]]; then
  MENTION="[To:${NOTIFY_TO_ACCOUNT_ID}] "
else
  MENTION=""
fi

BODY="${MENTION}[info][title]${EMOJI} Claude Code ${STATUS} (${NOW})[/title]セッションID: ${SESSION_ID}
作業ディレクトリ: ${CWD}
停止理由: ${STOP_REASON}

📝 最後のメッセージ:
${LAST_MESSAGE}[/info]"

HTTP_CODE=$(curl -s -o /tmp/chatwork_response.json -w "%{http_code}" \
  -X POST \
  -H "x-chatworktoken: ${CHATWORK_API_TOKEN}" \
  --data-urlencode "body=${BODY}" \
  "https://api.chatwork.com/v2/rooms/${CHATWORK_ROOM_ID}/messages")

if [[ "$HTTP_CODE" -eq 200 ]]; then
  MESSAGE_ID=$(python3 -c "import json; print(json.load(open('/tmp/chatwork_response.json')).get('message_id',''))" 2>/dev/null || echo "")
  echo "[notify_chatwork] Chatwork 送信成功 (message_id=${MESSAGE_ID})" >&2
else
  echo "[notify_chatwork] Chatwork 送信失敗 HTTP=${HTTP_CODE}" >&2
fi

exit 0
```

### 5. 環境変数の設定

```bash
export CHATWORK_API_TOKEN="your_api_token"
export CHATWORK_ROOM_ID="123456789"
export NOTIFY_TO_ACCOUNT_ID="9876543"  # 省略可
```

`.env` ファイルに書いて `source .env` でもOK。その場合は `.gitignore` に追加を忘れずに。

## やってみた感想

### 良かった点

- **実装が本当に簡単**：シェルスクリプトとJSONを書くだけ。インフラ知識がなくても30分以内で完成しました
- **停止理由の分岐が便利**：`end_turn`（正常完了）と `max_tokens`（トークン上限）で通知の絵文字が変わるので、ひと目でわかります
- **Claudeの最後のメッセージが通知に含まれる**：「何が終わったか」がChatworkを見るだけで把握できる。画面に戻らなくていい

### 惜しかった点

- **transcript のパースが実験的**：`transcript_path` の JSON フォーマットは公式ドキュメントに詳しく書かれていないため、将来の仕様変更で壊れる可能性があります。ここは fallback を入れて対応しました
- **Chatwork APIは `--data-urlencode` が必要**：日本語の通知テキストにURLエンコードが必要で、`-d` だと文字化けします。ここで15分ハマりました

### 業務で使えるか？

**使えます**、というか既に使っています。

特に効果があるのは、**長い分析タスクを Claude Code に任せるとき**。「このCSVデータを整形してサマリーを作ってほしい」とか「このコードのテストを全部書いてほしい」みたいな5〜10分かかる作業中に、別の仕事を並行できるようになりました。

チームのグループルームに通知するようにすれば、「Claudeが今何をやっているか」を共有することもできます。コンサル業務でクライアント向けの資料を自動生成している場合など、メンバー全員がステータスを把握できるのは地味に便利です。
