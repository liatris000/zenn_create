# 記事執筆 skill

zenn 記事の本文を書く時に守るルール。詳細は `docs/article-style-guide.md` を参照。

## 発動タイミング

- Day 1 で記事の下書きを作成する時
- Day 2 で実装の本文を書く時
- Day 3 でセルフレビューする時
- 既存記事のメンテナンス(タイポ修正等)を行う時

## 発動時に必ず確認すること

### 禁止表現リスト(冒頭で必ず確認)

リード文に以下を使わない:

- 「〜と感じたことはないでしょうか」
- 「〜で悩んでいる方も多いのではないでしょうか」
- 「結論から言うと、〜」
- 「今回は〜について解説します」

文末に以下を使わない:

- 「いかがでしたか」
- 「ぜひ試してみてください」
- 「参考になれば幸いです」

構成に以下を毎記事置かない:

- 「良かった点 / 惜しかった点 / 業務での活用イメージ」(毎記事使うとテンプレ化する)
- 「まとめ」(自然に終われるなら不要)
- 「こんな方に試してほしいです」(締めには使わない)

### 必須セクション

- **データアナリスト視点**(2〜3 文): 詳細は `docs/article-style-guide.md` の「やること」セクション参照
- **ハマった点 / 思考プロセス**: 自然に織り込む(明示見出しを使わない)
- **動くコード**: コピペで動く完全形、言語指定あり

### 情報漏れ対策

- 業務文脈・関係者・取引先・パートナー・提携先を出さない
- 本名「平野翔斗」を記事本文内に出さない(プロフィール表示は OK)
- フィクションの数値を書かない
- 詳細は `business-profile/policies/disclosure-rules.md` と `docs/cycle-overview.md` 参照

### 文字数

- 1500〜3000 字を基本
- 範囲外なら調整 or 分割を検討

### 構成パターンの選択

題材に応じて以下のパターンから選ぶ(または組み合わせる)。
毎記事同じパターンを使わない。

- パターン 1: 実装編
- パターン 2: 比較・検証編
- パターン 3: 概念解説編

詳細は `docs/article-style-guide.md` の「構成パターン」セクション。

## セルフレビュー時のチェックリスト

Day 3 のセルフレビューで以下を確認:

- [ ] リード文がテンプレ表現を避けている
- [ ] 文末がテンプレ表現を避けている
- [ ] データアナリスト視点が 2〜3 文入っている
- [ ] ハマった点 / 思考プロセスが自然に織り込まれている
- [ ] 業務コンテクストが出ていない
- [ ] 本名が記事本文内にない
- [ ] フィクション数値がない
- [ ] コードが動く形(言語指定あり)
- [ ] 文字数 1500〜3000 字

詳細は `docs/article-style-guide.md` のチェックリストを参照。
