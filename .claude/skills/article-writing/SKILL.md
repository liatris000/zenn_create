# 記事執筆 skill

zenn 記事の本文を書く時に守るルール。詳細は `docs/article-style-guide.md` を参照。

## 発動タイミング

- Day 1 で記事の下書きを作成する時
- Day 2 で実装の本文を書く時
- Day 3 でセルフレビューする時
- 既存記事のメンテナンス(タイポ修正等)を行う時

## 発動時に必ず確認すること

### 禁止表現リスト(冒頭で必ず確認)

リード文に以下を使わない:

- 「〜と感じたことはないでしょうか」
- 「〜で悩んでいる方も多いのではないでしょうか」
- 「結論から言うと、〜」
- 「今回は〜について解説します」

文末に以下を使わない:

- 「いかがでしたか」
- 「ぜひ試してみてください」
- 「参考になれば幸いです」

構成に以下を毎記事置かない:

- 「良かった点 / 惜しかった点 / 業務での活用イメージ」(毎記事使うとテンプレ化する)
- 「まとめ」(自然に終われるなら不要)
- 「こんな方に試してほしいです」(締めには使わない)

### 必須セクション

- **データアナリスト視点**(2〜3 文): 詳細は `docs/article-style-guide.md` の「やること」セクション参照
- **ハマった点 / 思考プロセス**: 自然に織り込む(明示見出しを使わない)
- **動くコード**: コピペで動く完全形、言語指定あり

### 原文保全ルール(識別子を丸めない)

推敲・要約・リライトの際、具体的な識別子を「○○等」「新しめのバージョン」のような
抽象表現に丸めない。実運用のノウハウは丸めると価値がゼロになる。読者にとって重要
なのは差分であって、抽象化された一般論ではない。「Node の新しめのバージョンで」
ではなく「Node 22.11.0 で」と書けるかどうかが再現性を分ける。

**要約・抽象化を禁じるトークン**(原文の表記をそのまま残す):

- ファイル名・パス(例: `.claude/settings.json`)
- バージョン番号(ランタイム / ライブラリ / Docker イメージ。例: `Node 22.11.0`)
- 設定キー名(例: `delete_branch_on_merge`)
- 環境変数名(例: `NG_WORDS`)
- エラーメッセージの文言(原文ママ。翻訳・意訳・省略しない)
- コマンドのフラグ(例: `--no-merged`)

**要約してよい範囲**: 設計判断の背景、一般原則の解説、Why の説明。
ただしその中でも、特定の識別子に言及する箇所では識別子を省略しない。

**引用形式(またはコードブロック)で原文のまま残す文脈**:

- 注意 / caveat / known issue / workaround
- 「◯◯の場合は××に注意」のような条件付きノウハウ

これらは要約すると条件や前提が落ちて誤った一般化になるため、要約対象にしない。

### 情報漏れ対策

- 業務文脈・関係者・取引先・パートナー・提携先を出さない
- 本名「平野翔斗」を記事本文内に出さない(プロフィール表示は OK)
- フィクションの数値を書かない
- 詳細は `business-profile/policies/disclosure-rules.md` と `docs/cycle-overview.md` 参照

### 文字数

- 1500〜3000 字を基本
- 範囲外なら調整 or 分割を検討

### 構成パターンの選択

題材に応じて以下のパターンから選ぶ(または組み合わせる)。
毎記事同じパターンを使わない。

- パターン 1: 実装編
- パターン 2: 比較・検証編
- パターン 3: 概念解説編

詳細は `docs/article-style-guide.md` の「構成パターン」セクション。

## セルフレビュー時のチェックリスト

Day 3 のセルフレビューで以下を確認:

- [ ] リード文がテンプレ表現を避けている
- [ ] 文末がテンプレ表現を避けている
- [ ] データアナリスト視点が 2〜3 文入っている
- [ ] ハマった点 / 思考プロセスが自然に織り込まれている
- [ ] 業務コンテクストが出ていない
- [ ] 本名が記事本文内にない
- [ ] フィクション数値がない
- [ ] コードが動く形(言語指定あり)
- [ ] 文字数 1500〜3000 字
- [ ] 識別子(ファイル名・パス・バージョン番号・設定キー・環境変数・エラーメッセージ・フラグ)が「○○等」に丸められていない(原文保全ルール参照)
- [ ] 注意・caveat・条件付きノウハウが要約で条件を落とされていない

詳細は `docs/article-style-guide.md` のチェックリストを参照。
